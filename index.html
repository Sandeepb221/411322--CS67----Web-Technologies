Profile

I am Sandeep MOLLETI. This portfolio demonstrates my practical understanding of core front-end web technologies. The aim is to present work in a way that is academically clear, easy to evaluate, and suitable for viva discussion. The website is intentionally framework-free to show direct competency in HTML, CSS, and JavaScript.

Design rationale

A technical resume theme is selected because it prioritises readability, structured information, and assessment clarity. The interface uses restrained colour accents and consistent spacing to maintain a professional tone suitable for academic submission.

What the assessor should notice

Each section maps to a learning outcome: semantic structure, responsive layout, controlled interactivity, validation, and testing evidence. Interactive features are intentionally simple but meaningful, which supports reliability and clear explanation during the presentation.

Skills (Accordion Demonstration)

This section uses an accordion component to demonstrate JavaScript event handling and DOM state changes. Clicking each header expands or collapses the related content. The written content is coursework-focused and describes how each skill area is applied in this portfolio.

Semantic tags are used to structure the website, including header/navigation, main content sections, and a footer. Proper heading hierarchy improves readability and supports assistive technology. Labels are applied to form controls to ensure the interface remains accessible and academically appropriate.

This directly supports CS67 requirements related to correct document structure, accessibility principles, and maintainable code organisation.

CSS is used to create a consistent visual system: reusable cards, buttons, spacing rules, and typographic hierarchy. Grid and flexible layouts are used so the site adapts to mobile screens without breaking content flow. The design remains readable and stable across different devices.

This is assessed through responsiveness, design coherence, and appropriate use of layout techniques.

JavaScript powers the accordion behaviour, mobile navigation toggling, scroll-spy highlighting, and contact form validation. These features demonstrate event listeners, DOM selection, conditional logic, and dynamic UI updates without reliance on external libraries.

The code is intentionally readable and structured so it can be explained clearly in a viva.

Evidence of Coursework Requirements

This section provides assessable evidence aligned with the CS67 learning outcomes. Each evidence item explains what was implemented, why it was implemented, and how it supports usability and academic marking. The accordion format keeps content structured and readable.

The portfolio uses semantic HTML sections and navigation anchors so users can move between content areas quickly. This supports both user experience and assessment review, since an assessor can locate evidence without scrolling through unrelated content.

The navigation is paired with scroll-spy highlighting to show active position, demonstrating a practical JavaScript enhancement that improves clarity during review.

Content is displayed using a card-based layout that adapts from multi-column desktop views to stacked mobile views using media queries. Spacing and typography rules are consistent, which supports readability and gives the site a professional academic tone.

The responsive design is verified in the testing section using manual checks across multiple screen widths.

JavaScript interaction is implemented in four areas: mobile menu toggle, accordion behaviour, scroll-spy navigation, and contact form validation. Validation checks help ensure data quality by verifying name length, email format, and message length before acceptance.

These are assessed as functional interaction features that demonstrate DOM manipulation and user feedback.

Testing Summary

This testing summary documents how the portfolio was checked during development. Testing focuses on responsive behaviour, interaction correctness, and basic accessibility, which supports both reliability and viva justification.

Responsiveness checks

The website was tested using browser developer tools at multiple widths (desktop, tablet, and mobile). The layout was verified to ensure cards stack correctly, text remains readable, and navigation stays accessible without horizontal scrolling.

JavaScript behaviour

Accordion panels were tested for correct expand/collapse behaviour, ensuring only the intended panel opens at a time. The scroll-spy navigation highlight was verified while scrolling. The mobile menu toggle was tested for open/close stability across device sizes.

Form validation

The form was tested with empty fields, invalid emails, and short messages to confirm error messages display clearly. Valid entries were tested to confirm the success message appears and the form resets for demo purposes.

Contact (Validation Demonstration)

This form is included for coursework demonstration. It validates input on the client-side and provides clear feedback messages. It does not submit data to a server.
